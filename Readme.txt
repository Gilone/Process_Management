本系统使用Python实现，可以通过解释器直接运行。

共包含了五个模块：
Test Shell 负责用户的交互，它调用系统的 Core 核心模块。
Core 模块是进程和资源的管理，调用 Ready List，PCB 和 RCB 模块。
Ready List，PCB 和 RCB 模块相互独立，各自对自己的链表进行管理。

PCB：该模块负责进程控制块的实现，以及子进程链表的设计和管理。
RCB：这个模块是为了实现资源控制块和阻塞队列的设计和管理。每个资源有对应数量的资源数，共有四种资源。每个资源有响应的阻塞队列，阻塞队列的节点指向相应的进程控制块。
ReadyList：这里实现就绪队列的管理，每个节点指向一个进程控制块，这个链表相当于管理所有在 ready 状态和 running 状态的进程控制块。
Core：该模块为最核心的管理部分，调用 PCB,RCB 和 ReadyList 模块。包含大量的方法，用于实现进程和资源的管理。
Test Shell：该模块首先调用 Core 模块中的 ini 函数，进行根进程的初始化。然后打印相关的提示信息，告知用户可以输入的内容。然后进入一个无限的循环。在这个循
环里，重复提示用户输入命令，然后进入对应的分支选项。调用 Core 中对应的
函数。
